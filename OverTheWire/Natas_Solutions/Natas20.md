The vulnerability of this problem  
1.   
```php
foreach($_SESSION as $key => $value) {
        $data .= "$key $value\n";
```  
Data is stored simply as \n delimeter. -> So we can inject \n delimeter to $value, we can make two raw data

2.  
```php
 foreach(explode("\n", $data) as $line) {
    $parts = explode(" ", $line, 2);
    if($parts[0] != "") $_SESSION[$parts[0]] = $parts[1];
```  
When reading session, server reads each line, and recognize first part as key, second part as value.  

Exploitation:  
?name=test%0aadmin%201 -> ?name=test\nadmin 1 -> {test:[Random Array]}, {admin:1}  
If refreshing the page, myread() read that datas, set the value of admin 1    
So satisfying the condition of print_crudentials, we can get a password for next level.
